# Application Layout

Source: [DeepWiki `5.1-application-layout`](https://deepwiki.com/tmoroney/auto-subs/5.1-application-layout)

## Purpose

This page documents the top-level UI composition and how global providers, theme behavior, and responsive layout are organized.

## Main Elements

- root app component
- global providers and contexts
- desktop and mobile layout branches
- top-level routing and page containers

## Key Outcome

The layout ensures standalone and Resolve-connected workflows can share most UI infrastructure while still supporting mode-specific behavior.
